# Alembic Database Migrations

This directory contains Alembic database migration scripts for Revellio.

## Structure

- `env.py` - Migration environment configuration (loads models and settings)
- `script.py.mako` - Template for generating new migration files
- `versions/` - Directory containing all migration files (chronological order)

## Common Commands

See `justfile` for convenient commands:

```bash
# Create a new migration (autogenerate from model changes)
just migrate "description of changes"

# Apply all pending migrations
just upgrade

# Rollback one migration
just downgrade

# Show migration history
just migration-history

# Show current migration version
just migration-current
```

## Manual Alembic Commands

If you need more control:

```bash
# Create a new migration manually
alembic revision -m "description"

# Create a migration with autogenerate
alembic revision --autogenerate -m "description"

# Apply migrations
alembic upgrade head

# Rollback one migration
alembic downgrade -1

# Rollback to specific revision
alembic downgrade <revision_id>

# Show current version
alembic current

# Show history
alembic history
```

## Migration Workflow

1. **Modify models** in `db/models.py`
2. **Create migration**: `just migrate "add new column to users"`
3. **Review generated migration** in `alembic/versions/`
4. **Apply migration**: `just upgrade`
5. **Commit** both the model changes and migration file to git

## Tips

- **Always review autogenerated migrations** - They're not always perfect!
- **Test rollback** before deploying to production
- **Never edit applied migrations** - Create a new migration instead
- **Keep migrations small** - One logical change per migration
- **Write descriptive messages** - Future you will thank present you
